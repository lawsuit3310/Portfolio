게임 개요

Merge 게임


 2024년 가천대학교 겨울 계절학기 팀 프로젝트를 진행하게 되어, Merge 게임을 주제로 게임을 개발하였습니다.

 턴의 경과에 따라 감소하는 허기와 갈증 수치들을 "카드" 를 사용해 회복하며 진행하는 게임으로, 같은 등급의
카드들을 합쳐 더욱 높은 등급의 카드를 만들어내며 나타나는 다양한 조합의 카드들을 모으는 것이 주 목표인
게임입니다.


1주차 개발 진행 내용

게임 턴 시스템 개발 (1/4)

 게임의 중심이 될 턴 시스템을 제작하였습니다. 턴의 경과에 따라 Encounter가 발생하거나, ' 현재 턴 동안 카드를
생성할 수 있는 횟수 ' 를 의미하는 AP(행동력) 수치가 회복 되거나, 허기 / 갈증 수치가 감소하기도 하는 등, 게임의
진행에 있어 필수적인 요소이기에 가장 먼저 그 구조를 세우고 Interface를 맞추는 작업을 진행하였습니다.

1주차 개발 진행 내용

카드 조작, 조합 관련 기능 구현 (1/5)

 앞선 턴 시스템에 이어, 게임 진행에 있어 핵심이 되는 요소인 "카드" 와 그에 관련된 시스템을 개발하였습니다.


  • 카드 조작
    • Unity API의 OnMouseUp / OnMouseDrag / OnMouseDown를 활용하여
      제작하였습니다.

  • 카드 매니저
    • 카드를 관리하기 위한 클래스로, 카드의 생성 및 파괴를 실행해주는 Interface를
      제공합니다.
    • C# API에서 제공하는 Linq를 활용해 현재 생성되어있는 카드들을 대상으로 Query를
      진행하여 필요한 카드만 취합하여 전달 받을 수 있습니다.

  • 카드 합치기
    • 카드를 Merge 시키는 것은 게임 내 특정한 위치(Merging Area)에 합칠 카드를 내려놓아
      서로 겹치는 것으로 진행됩니다.
    • 카드를 Merging Area가 아닌 다른 곳에 내려놓는다면, 후술할 카드 그룹을 생성하는
      과정으로 넘어갑니다.

  • 카드 그룹
    • 카드를 정렬 하기 위해 제작된 클래스 입니다.
    • 카드를 서로 겹칠 경우 생성되며, AddCard, RemoveCard 등의 Interface를 제공하고
      있습니다.


1주차 개발 진행 내용

xlsx 파일 역직렬화 (1/8)

 카드의 실제 데이터 및 카드를 Merge 할 때의 조합 방식 등의 데이터를 xlsx 파일로 저장하고 이를 역직렬화하여
실제 게임에 적용 시키도록하는 기능을 개발하였습니다.

특수한 조합 방식의 구현 (1/8)

 일반적인 경우, 같은 종류, 같은 등급의 카드만 서로 합치는 것이 가능하지만, 몇몇 카드의 경우 완전히 서로 다른
두 카드를 합치는 것으로 만들어지는 경우가 있어 해당하는 기능을 구현하였습니다.

적 캐릭터의 구현 (1/9)

 게임이 지나치게 루즈 할 수 있다는 내부 의견으로 적 캐릭터를
추가하게 되었습니다.

적 캐릭터는 턴이 경과 할 때 랜덤하게 출현하며, 출현 시
플레이어의 식량 카드를 향해 이동, 공격하도록 설정되어 있습니다.

플레이어는 플레이어가 가진 자원 카드를 활용하여 이를 처치 할 수 있습니다.

1주차 개발 진행 내용

카드의 "정렬" 기능 구현 (1/9)

 게임을 진행함에 있어 현재 생성되어있는 카드들을 한 곳에 모아놓고
관리하고 싶은 경우가 있을 수 있습니다.

 그런 경우에 사용할 수 있도록 카드를 종류, 티어에 따라 분류하여
한 곳에 정렬 할 수 있는 기능을 구현 하였습니다.


카드 조작 관련 버그 수정 (1/10)

 카드를 조작할 때 발생하는 버그를 수정하였습니다.

게임 재시작 기능 (1/10)

 게임 오버 되었을 경우 게임을 다시 시작할 수 있게하는 기능을 추가하였습니다.

카드 분해 기능 구현 (1/10)

 2티어 이상인 카드들을 분해하여 하위 티어인 카드 2장을 획득할 수 있도록 하는 기능을 개발하였습니다.

1주차 개발 진행 내용

게임 내 공지 시스템 추가 (1/11)

 게임을 진행하면서 플레이어에게 무조건 알려야하는 정보가 있을 수 있습니다. 그런 상황에서, 플레이어에게 이를
확실히 알리기 위해서 공지를 띄울 수 있게하는 시스템을 개발하였습니다.

 또한 유니티 API의 버튼 클래스를 이용해 구현되어 공지를 클릭하였을 때 동작할 내용을 대리자로 전달, 이를
onclick에 추가하는 방식으로 구현하는 식으로 상황에 따라 다르게 동작 할 수 있도록 구현하였습니다.


Yaml 파일을 사용한 데이터 관리 (1/11)

 게임 플레이 도중 획득한 적이 있는 카드들에 대한 정보를 yaml 파일을 활용해 저장하여 추후 이를 기반으로
"도감 시스템" 을 제작할 수 있도록 하였습니다.

스테이터스를 표시하는 창 추가 (1/12)

  허기, 갈증, 행동력 등의 수치를 표시하여 주는 UI를 제작, 실제 데이터와 연동하여 게임에 적용시켰습니다.

카드가 겹쳐져 있는 상태에서도 합쳐질 수 있도록 하는 기능 추가 (1/12)

 카드가 이미 겹쳐져 있는 상태에서 Merging Area에 가져다 두었을 경우에도 Merge 가 진행 될 수 있도록
하였습니다.
